# 🤖 AI投資司令塔 - CIO Prototype システム設計書

本ドキュメントは、AIを活用した株式分析システム「CIO Prototype」の現状の設計と仕様をまとめたものです。

## 1. システム概要
銘柄コード（日本株・米国株）を入力するだけで、財務データ、テクニカル指標、マクロ環境、および有価証券報告書（有報/10-K）を統合的に解析し、プロフェッショナルな投資レポートと4軸スコアカードを自動生成するシステムです。

### 主要な目標
- **市場のバグの発見**: 財務数値と定性情報をAIで照合し、市場が見落としている本質的価値を特定する。
- **CIOレベルの意思決定補助**: 複数の判断指標を統合し、BUY/WATCH/SELLのシグナルを提示する。
- **分析の自動化**: yfinance、EDINET、SEC、Gemini/Groqを連携させた自動パイプライン。

## 3. コンポーネント詳細

### 3.1 オーケストレーター (`main.py`)
システムの司令塔として、以下のフローを制御します：
1. 分析対象銘柄の基本データ取得。
2. マクロ環境の判定。
3. 比較対象銘柄（競合）の自動選定とデータ取得。
4. 定性データ（有報/10-K）の取得と解析。
5. DCF法による理論株価の算出。
6. 4軸スコアカードの生成。
7. AIによる総合投資レポートの生成。
8. 結果の保存と通知。

### 3.2 4軸スコアリングエンジン (`analyzers.py`)
定量的・定性的なデータを4つのレイヤーで評価します（0.0〜10.0点）。

#### Layer 1: Fundamental (企業の地力)
長期的な企業価値を生み出す「稼ぐ力」と「財務の健全性」を評価します。

| 評価項目 | 10点 (S評価) | 7点 (A評価) | 4点 (B評価) | 0点 (C評価) | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **ROE** | 基準値の2倍以上 | 基準値以上 (通常10%) | 5%以上 | 0%以下 | 資本効率の高さ |
| **営業利益率** | 基準値の2倍以上 | 基準値以上 (通常15%) | 5%以上 | 0%以下 | 本業の収益力 |
| **自己資本比率** | 基準値の2倍以上 | 基準値以上 (通常40%) | 20%以上 | 20%未満 | 財務の安全性 |
| **CF品質** | 1.5倍以上 | 1.0倍以上 | 0.5倍以上 | 0.5倍未満 | 営業CF / 純利益 |
| **R&D比率** | 15%以上 | 8%以上 | 3%以上 | 3%未満 | 将来への投資 (セクター加重あり) |

#### Layer 2: Valuation (投資の割安度)
現在の株価が企業価値に対して割安かを評価します。

| 評価項目 | 10点 (超割安) | 7-8点 (割安) | 4-5点 (適正) | 1-3点 (割高) | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **PER** | 基準値の0.5倍以下 | 基準値以下 (通常15倍) | 基準値の2倍程度 | 2倍超 | 利益倍率 |
| **PBR** | 基準値の0.5倍以下 | 基準値以下 (通常1.0倍) | 3倍程度 | 5倍超 | 資産倍率 |
| **配当利回り** | 5%以上 | 3%以上 | 1.5%以上 | 1.5%未満 | インカムゲイン |
| **目標株価乖離** | +30%以上 | +15%以上 | 0%以上 | 0%未満 | アナリストコンセンサス比較 |
| **DCF乖離** | +30%以上 | +15%以上 | 0%以上 | 0%未満 | 理論株価との比較 (安全域) |

#### Layer 3: Technical (エントリータイミング)
「安く買う」ための最適なタイミングをテクニカル指標で判断します。

| 評価項目 | 9-10点 (買い時) | 7点 (やや買い) | 5点 (中立) | 1-3点 (売り/過熱) | 備考 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **RSI (14)** | 30以下 | 45以下 | 45-55 | 70以上 | 売られすぎ/買われすぎ判定 |
| **MA25乖離率** | -10%以下 | -3%以下 | ±3%以内 | +10%以上 | 短期トレンドからの乖離 |
| **MA75乖離率** | -15%以下 | -5%以下 | ±5%以内 | +15%以上 | 中期トレンドからの乖離 |
| **BB位置** | 下限バンド付近 (10%) | 30%以下 | 中央 (30-70%) | 上限バンド付近 (90%) | ボリンジャーバンド位置 |
| **出来高倍率** | 2.0倍以上 (注目) | 1.2倍以上 | 0.8-1.2倍 | 0.8倍未満 | 平常時との比較 |

> [!TIP]
> **セクター別補正**: セクター（Technology, Financial等）に応じて、評価基準（閾値）と各軸の重みを自動調整します。

### 3.3 マクロ環境判定 (`macro_regime.py`)
VIX指数、米10年債利回り、ドル円、原油価格から現在の相場環境（Regime）を特定します。
- `RISK_ON`, `RISK_OFF`, `RATE_HIKE`, `RATE_CUT`, `NEUTRAL`
- 環境に応じて、スコアカードの重みを動的に変更（例：金利上昇時はグロース株のValuation重みをアップ）。

### 3.4 DCF理論株価モデル (`dcf_model.py`)
フリーキャッシュフロー（FCF）の履歴に基づき、将来の成長シナリオを予測して理論株価を算出します。
- 3つのシナリオ（Bull / Base / Bear）を提示。
- 安全域（Margin of Safety）と上昇余地を可視化。

### 3.5 AIエンジン (`data_fetcher.py`)
Gemini API (3.0 Flash/Pro) を主力とし、レート制限時には Groq (Llama 3) へ自動フォールバックする堅牢な実装です。

---

### 3.6 バックテストモジュール (`src/backtester.py`)
過去の特定日を指定して分析を実行し、その後の価格推移と比較することで戦略の有効性を検証します。
- **バックテスト・解像度の動的変更**: 
    - **Long（長期）**: 月次ループで効率的にシミュレーション。
    - **Short（短期）**: 日次ループを採用し、月中の利確・損切りを分単位（日次単位）で正確に追跡。
- **取引コスト（スリッページ・手数料）**: `config.json` の `execution_cost_bps` に基づき、往復コストを資産価値に反映することで、現実的な利益率を算出。
- **データキャッシュ機構**: `data/cache/` にAPIレスポンスを保存し、同一銘柄の再分析やバックテストの高速化、およびAPIレート制限の回避を実現。
- **パフォーマンス計測**: トータル収益率、ベンチマーク比較、アルファ（市場超過利回り）を算出。
- **売買戦略オプション**:
    - **Long（長期戦略）**: 割安・優良銘柄をBUYシグナルで購入し、SELLシグナルが出るまで長期保有。
    - **Short（短期戦略）**: テクニカル指標を重視し、+5%で利確、-3%で損切りを行う回転売買ロジック。

## 4. UI/UX

### 4.1 Streamlit Dashboard (`app.py`)
- **インタラクティブ分析**: ブラウザ上で銘柄を入力し、リアルタイムで分析を実行。
- **モジュール構成**: `src/` 配下のパッケージ化されたモジュールを利用し、保守性を向上。
- **履歴閲覧**: 過去の分析結果をサイドバーから即座に呼び出し。

### 4.2 Web Intelligence Dashboard (`web/index.html`)
- **リソース分離**: HTML/JS/CSS を `web/` ディレクトリに分離し、開発の見通しを改善。
- **静的配信**: `serve.py` を通じて `web/` フォルダを起点とした配信を実施。

## 5. データ連携と拡張性

- **Google Sheets**: 分析結果をスプレッドシートに自動書き出しし、ポートフォリオ管理に活用。

- **Config**: `config.json` により、スコアリングの閾値やセクター定義をコード変更なしでデ正可能。
