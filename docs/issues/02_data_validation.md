# Issue: データ品質の向上とバリデーション強化

## 概要 (Overview)
各種データソース（価格データ、財務・ファンダメンタル指標等）から取得した値が、不正な形式（NaN、極端な外れ値など）であった場合に、後続のスコアリング処理やダッシュボード表示に悪影響を及ぼすのを防ぐ。

## 背景・課題 (Background / Problem)
- 現在、取得データの中に `NaN` (Not a Number) や `Infinity` が混入した場合、計算ロジック（例: DCFモデルや4軸スコア算出）内でエラー（定性スコア0.0やAnalysis Failure）を引き起こす原因となっている。
- 株式分割のデータ補正漏れなどにより、株価や指標が前日比で非連続的（例えば前日比±50%超など）になるケースがあり、これが誤ったシグナル判定（割安/割高判定の一時的な誤認など）を生む。

## 修正方針 (Proposed Solution)

### 1. DataFrame, NaN のガード節実装
- `src/data_fetcher.py` でデータを取得・整形した直後に、欠損値（NaN）チェックを入れる。
- 必須指標（終値、PER、ROEなどスコアリングに直結するもの）がNaNの場合は、分析対象から除外（スキップ）するか、安全なデフォルト値へのフォールバック（例：業界平均）を検討する（基本は除外とする）。

### 2. 異常値（Outlier）の検知ロジック追加
- 「株価の前日比変動率」や「主要財務指標（利益率など）の急変」をチェックするロジックを実装。
- 変動率が一定の閾値（例: ±50%以上など）を超えた場合は、株式分割やデータ提供元のエラーの可能性が高いため、Warningをログ（および `System_Log` シート）に出力し、該当データによる取引判断を一時保留にする。

### 3. スプレッドシート書込前の最終サニタイズ
- `src/sheets_writer.py` において、書き込むデータリスト（またはJSON）内に `NaN`, `Inf`, `-Inf` が含まれていないか検証し、Google Sheets API が受け付けられる形式（`null` など）に変換・置換する処理を追加。

- [ ] `dcf_model.py` の入力データ（成長率、現在価値など）のNaNハンドリング修正
- [ ] 異常値（前日比±50%超など）検知用バリデーション関数の作成

## 影響範囲 (Impact)
