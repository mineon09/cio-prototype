# Issue: ダッシュボードUIとフロントエンドの改善

## 概要 (Overview)
システムが出力するJSONファイル（`results.json` など）を元に、ブラウザから分析状態や最終的な投資判断（シグナル）を視覚的に一望できるダッシュボードの改善を行う。

## 背景・課題 (Background / Problem)
- 現在のダッシュボード（`serve.py` や `index.html` 等）は基本的な情報表示に留まっており、CIO（最高投資責任者）の視点での一覧性や、マクロ環境（Regime）と個別銘柄シグナルとの連動状況が分かりにくい。
- エラーが発生した場合（NaNやパース失敗等）の表示が不十分で、ユーザーにとって「計算中」なのか「失敗」したのか区別がつきにくい。

## 修正方針 (Proposed Solution)

### 1. マクロ環境（Regime）の可視化
- 画面上部（ヘッダー直下など）に、現在のマクロレジーム（例: RISK_ON, YIELD_INVERSION）をデカデカと表示し、全体の投資スタンスを一目で理解できるようにする。

### 2. 銘柄ステータスのリッチ化
- NaNエラーや外れ値により分析がスキップされた銘柄については、単に非表示にするのではなく、「Warning (Invalid Data)」等のステータスバッジを表示し、理由をツールチップで確認できるようにする。
- 4軸スコア（Fundamental, Valuation, Technical, Qualitative）をレーダーチャートやプログレスバー等で視覚的に比較しやすくする。

### 3. エラー・システムログの表示
- `System_Log` シートに書き込む内容と同様のサマリー（直近の実行ステータス、エラー件数など）をダッシュボード上でも確認（あるいはリンク経由で参照）できるようにする。

## タスクリスト (Tasks)
- [ ] ダッシュボード (`index.html`, `dashboard.js`, `dashboard.css`) にマクロレジーム表示領域を追加
- [ ] 銘柄カード（またはテーブル行）にエラー状態/Warningステータスのバッジ表示を追加
- [ ] 4軸スコアの視覚的コンポーネント（チャート等）の導入
- [ ] データ再フェッチ時のローディングステータスの改善

## 影響範囲 (Impact)
フロントエンドディレクトリ (`web/` または `index.html` 類) および ローカルサーバー (`serve.py`)。

## 備考 (Notes)
UXの観点から、不要な再読み込みを防ぐためのポーリングやWebSocket化も将来的な視野に入れるが、まずは静的な JSON 読み込みのリッチ化に留める。
